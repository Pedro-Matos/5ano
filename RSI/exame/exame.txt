3. O que é um Dicom persistent object e falar dele

An instance of a data object as defined by [DICOM PS 3.3] that has been allocated an unique identifier in the format specified for SOP Instance UID in [DICOM PS 3.3] 
and has been chosen as an object to be saved securely for some period of time. Within the DICOM Standard, a DICOM Persistent Object
 is referred to as a Composite Service Object Pair (SOP) Instance.

4. PACS na cloud (o que é, pa que serve, vantagens e desvanstagens)

Ao utilizarmos um serviço PACS na Cloud, estamos também a promover a partilha de informação entre instituições, 
de forma a criar um PACS Regional.

Vantagens:
	-> Redução da infraestrutura tecnológina no lado organizacional
	-> Redução da manutenção do servidor
	-> Redução dos custos
	-> Redução nos gastos energéticos e da climatização do ar nos servidores
	-> Redução das licenças necessárias
	-> Acesso às aplicações e dados em todo lado e a qualquer hora
Desvantagens:
	-> Possibilidade de existência de problemas de privacidade e confidencialidade
	-> Fraca interoperabilidade com os protocolos existentes
	-> Atrasos no acesso aos dads/aplicações
